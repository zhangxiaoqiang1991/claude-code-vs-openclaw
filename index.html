核心差异：两种设计哲学

Claude Code 和 OpenClaw 看起来越来越像（都有 memory、skill、定时任务），但出发点完全不同。不是谁抄谁，而是合理的 Agent 系统该有的东西都在那，但底层架构哲学差异巨大。

核心一句话：Claude Code 是为"深度推理协作"设计的终端编码工具（工匠的锤子），OpenClaw 是为"7×24 自动化编排"设计的个人 Agent 网关（数字员工）。

OpenClaw 最近迭代了什么

由奥地利开发者 Peter Steinberger 于 2025 年 11 月创建，已成为 GitHub 增长最快的开源项目之一（36 万+ Star）。近 30-60 天重要更新：

🎤 13 个 TTS 语音供应商

  • Azure Speech、Volcengine、ElevenLabs v3
  • 角色级语音定制

🧠 可插拔上下文引擎

  • Lossless-Claw → SQLite + DAG 摘要
  • OOLONG 跑分 74.8（超 CC 的 70.3）

📊 OpenTelemetry 全栈可观测

  • 追踪模型调用、token、工具循环

🤝 Google Meet 插件

  • 直进会议、实时语音 loop

🎛 Task Control Plane

  • SQLite 任务账本，统一管理所有任务类型

🛡 安全加固

  • 语义化审批、fail-closed 安装
  • 内联 eval 检测

OpenClaw 的 24/7 守护进程

不是"一直在聊天"，而是"一直在线等你需要它"。

场景
异步任务编排
在 WhatsApp 说"帮我 review PR"，它在后台启动编码引擎做 review，做完推送结果。不需求开终端。
场景
跨系统自动化
查日历 → 订会议室 → 发确认邮件，一个指令串起多个系统。
场景
长时间后台任务
跑数据、爬资料、做研究，不用守着终端。
场景
IM 个人助理
Telegram / WhatsApp 直接对话，非技术用户也能用。

本质区别：Claude Code = 你主动找它干活；OpenClaw = 它一直在那等你找它。

记忆系统设计哲学：Push vs Pull

这是解释为什么 OpenClaw 耗 token 而 Claude Code 不耗的核心原因：

维度OpenClaw（Push）Claude Code（Pull）
启动时全量加载 SOUL+AGENTS+USER+MEMORY仅 CLAUDE.md + MEMORY.md 索引
对话中所有记忆一直在 context 里索引在 context，按需读取
上下文开销2-5 倍系统 prompt token~1K tokens
策略不知道问什么，全部装上膛知道在做什么，没必要全加载
优点任何问题都能答轻量、省 token、专注
缺点贵，噪声稀释 context需要 Agent 有判断力

两种哲学：OpenClaw 相信"给模型看更多，让它自己判断"，Claude Code 相信"让 Agent 自己判断需要什么"。

记忆写入原则：只记原则性的、可复用的方法论；不记日常项目、一次性任务。每条记忆都该是将来可复用的信号，不是对话存档。

7

Token 消耗分析

OpenClaw 的 token 消耗远高于 Claude Code，核心原因：

  1. 全量加载 — SOUL.md（~20K chars）+ AGENTS.md + USER.md + MEMORY.md 全塞入 system prompt
  2. Gateway 层开销 — IM 协议解析、路由转发，每层都加 token
  3. 模型无优化 — 无法像 Claude Code 为单一模型做深度上下文压缩
  4. 24/7 常驻 — idle 也在消耗资源

直观对比

每次请求的 system prompt token 消耗：

OpenClaw:     SOUL(~5K)+AGENTS(~3K)+USER(~1K)+MEMORY(~5K)
            = ~14K tokens

Claude Code:  CLAUDE(~1K)+MEMORY索引(~0.5K)
            = ~1.5K tokens

差异：约 5-10 倍

对你来说：每天 8-10 小时高强度使用，成本也远低于 OpenClaw 跑一个下午。你说过的——"在 OpenClaw 上面，再多的钱都扛不住"。

  USER.md     — 为谁服务（目的地）
  AGENTS.md   — 怎么做事（导航路线）

OpenClaw 是分散式多文件（职责清晰、可各自改写），Claude Code 是集中式（维护简单、不易遗漏）。

9

总结：什么场景用什么

✅ 选 Claude Code

  • 内容创作者 / 增长运营
  • 需要深度推理和高质量输出
  • 稳定 > 灵活，可控 > 无限可能
  • 每天用 8-10 小时，成本敏感
  • 不想要"半夜自己改代码"的 Agent

✅ 选 OpenClaw

  • 需要 7×24 待命的数字员工
  • 团队在多个 IM 平台上
  • 做 Agent 自进化的实验项目
  • 需要跨系统编排
  • 对 token 成本不敏感

强哥的选择：Claude Code + 自建 skill 体系 + auto memory + 定时任务 + 飞书 MCP。用 Claude Code 做内核，自己搭一套轻量级 OpenClaw-like 系统。稳定、省 token、不被锁定。

"一个工具好不好，不看它有多少功能，看你愿不愿意整天待在它里面。"
